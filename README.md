# bench-build — бенчмарк бандлеров Next.js

Сравнивает **rspack / webpack / turbopack** на одном и том же Next-приложении: время сборки, пик RAM, загрузку CPU и итоговый вес бандла. Один файл, без зависимостей.

## Что меряет

Для каждого движка — три уровня конфига:

- **base** — минимальный конфиг (дефолтный чанкинг) → сравнение самих движков;
- **clean** — чистый Next + textbook `splitChunks`/`cacheGroups` → «потолок правильных настроек»;
- **custom** — реальный `next.config.mjs` приложения (только если он есть).

Каждый вариант гоняется **cold** (чистый distDir) и **hot** (2-й прогон с кэшем). Метрики:

- **время** сборки;
- **пик RAM** (RSS всего дерева процессов);
- **CPU avg→peak** в «ядрах» (среднее по сборке → самое параллельное окно);
- **вес** бандла — суммарный RAW (`.js`+`.css` в `static`) + 3 самых тяжёлых чанка.

## Требования

- **Node 18+**, без внешних пакетов.
- Запускать **из корня репозитория**.
- Конфиг приложения — `next.config.mjs` (для `.ts`/`.js` тул прервётся, не трогая билд; без конфига — `base`/`clean` работают, `custom` пропускается).
- rspack-варианты включаются, только если установлен `next-rspack`.

## Запуск

```bash
node bench-build.mjs                 # все варианты, cold+hot
node bench-build.mjs --cold-only     # только cold (быстрее)
node bench-build.mjs --md            # + markdown-таблица для статьи
node bench-build.mjs --app=apps/web  # указать приложение явно
node bench-build.mjs --only=rspack,turbopack,rspack-custom
node bench-build.mjs --dry           # проверка детекта без сборки
```

## Как это безопасно

Билд идёт в **изолированный `.next-bench`** — рабочий `.next` и запущенный сервер не трогаются. На время прогона `next.config.mjs` подменяется временным и **восстанавливается** — в т.ч. по `Ctrl-C` (SIGINT/SIGTERM). Авто-детект: приложение (`apps/*`/корень), workspace-пакеты (`packages/*`, `libs/*`), наличие `next-rspack`.

## Важные оговорки при чтении цифр

- **turbopack игнорирует** `webpack()`/`cacheGroups` → его `clean`/`custom` = тот же дефолт.
- Вес здесь **RAW** — без пост-обработки (mangle/compress/dedup), не равен прод-бандлу.
- CPU **avg** занижен длинными серийными фазами Next (сбор page-data, RSC) — смотрите **peak** для реального параллелизма.
- Запускать **строго серийно** (тул так и делает): один билд за раз, без oversubscription.
